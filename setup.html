Full Guide

Full User Guide (Start to End)

Use this complete flow to run a quiz session from creation through final review.

1

Open SecureQuiz Home

Visit your hosted URL. Read the onboarding sections to understand how the platform works.

2

Go to Create Quiz

Click Create Quiz from the top navigation or CTA button to open the quiz creator page.

3

Enter Quiz Details

Fill quiz title, time limit, max tab switches, then add question text, options, and correct answer for each question.

4

Start Quiz and Generate Links

Click Start Quiz. SecureQuiz creates a unique student link and dashboard link for that quiz.

5

Share Student Link

Send only the generated student link. Students cannot open the quiz directly without this quiz ID link.

6

Student Attempt Flow

Students open the shared link, enter details, complete the quiz, and submit. Security events are tracked during the attempt.

7

Track Live Results

Open the dashboard link to monitor submissions, scores, timestamps, and integrity signals in real time.

8

End Quiz Session

Once all users finish, keep the dashboard as your final report. Start a new quiz from creator whenever needed.

Checklist

Quick User Checklist

A short reminder set for the moments right before launch.

  • Start from the SecureQuiz home page or open the creator workspace directly.
  • Share only the generated student link with participants, not the dashboard or creator page.
  • Keep the dashboard open during the live run so you can monitor results and integrity activity.
  • Verify time limit and question count before publishing so you do not need to rebuild the session.
Before You Share

Session Tips That Keep Things Smooth

These are small habits that make the quiz feel much more controlled once people join.

Tip 01 Open the dashboard on your own device first.

This gives you a clear view of incoming submissions before the first participant finishes.

Tip 02 Send instructions with the participant link.

Tell participants to stay focused on the quiz window so there is less confusion around integrity tracking.

Tip 03 Review unusual attempts with context.

Use the integrity signals as review input, not just as isolated numbers, when you assess results afterward.
