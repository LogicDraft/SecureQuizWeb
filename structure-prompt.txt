Here is the complete blueprint to transform your SecureQuiz from a static app into a fully functional, asynchronous SaaS platform (like Google Forms). 

### 📁 1. The Project Structure
You will need to reorganize your files to separate the Student experience from the Teacher experience. All files will share your existing `style.css` to maintain that premium dark glassmorphism look.

```text
/secure-quiz
│
├── index.html           # (Student) The main quiz interface
├── script.js            # (Student) Logic to fetch quiz & submit answers
│
├── create.html          # (Teacher) Form to create a new quiz
├── create.js            # (Teacher) Logic to push quiz to Firebase & get link
│
├── dashboard.html       # (Teacher) Table showing student results & cheat logs
├── dashboard.js         # (Teacher) Logic to fetch & display submission data
│
├── firebase-config.js   # (Shared) Your Firebase initialization keys
├── style.css            # (Shared) Your existing premium UI styles
└── report.html          # (Shared) The PDF marks card template
```

---

### 🗺️ 2. The Development Roadmap
Follow these steps in order so you don't get overwhelmed.

* **Phase 1: Firebase Setup (The Backend)**
    * Go to the Firebase Console and create a free project.
    * Enable **Firestore Database** (start in Test Mode so you can read/write without auth rules for now).
    * Copy your Firebase SDK config keys. Paste them into `firebase-config.js`.
* **Phase 2: The Creator Flow (`create.html`)**
    * Build a dark-themed form where a teacher types a Quiz Title, Time Limit, and pastes their JSON questions.
    * Write the JS to push this data to a `quizzes` collection in Firestore.
    * When successful, generate the shareable link: `yoursite.com/?quizId=[DOC_ID]`.
* **Phase 3: The Student Flow (`index.html`)**
    * Update your current `script.js`. Remove the hardcoded `questions.json` fetch.
    * Add logic to read the URL: `const urlParams = new URLSearchParams(window.location.search);`
    * Fetch the quiz data from Firestore using that ID.
    * When the student finishes, push their score and `state.tabSwitchCount` to `quizzes/[DOC_ID]/submissions`.
* **Phase 4: The Analytics Flow (`dashboard.html`)**
    * Build a page that reads the `quizId` from the URL.
    * Query the `submissions` sub-collection for that quiz.
    * Display the results in a clean HTML table so the teacher can see who cheated and who passed.
* **Phase 5: Deployment**
    * Push your new folder to GitHub and deploy it for free using Vercel, Netlify, or GitHub Pages.

### 🤖 3. The AI Master Prompt
Copy and paste this exact prompt into your AI coding assistant (like Cursor, Claude, or ChatGPT) to have it generate the foundational code for this entire system. 

```text
Act as a Senior Full-Stack Developer. I have an existing frontend Vanilla JS quiz application called "SecureQuiz" with a premium dark glassmorphism UI. I want to transition it into an asynchronous SaaS platform (similar to Google Forms) using Firebase Firestore (V9 Web SDK). 

I need the code for a system where a teacher creates a quiz, gets a unique URL, and students take it independently.

Please provide the HTML and JS for this new architecture, split into the following files:

### 1. Database Initialization (`firebase-config.js`)
- Provide the boilerplate Firebase V9 initialization code.

### 2. The Teacher "Create" Page (`create.html` & `create.js`)
- A beautiful dark-themed form to input: Quiz Title, Time Limit (minutes), and a text area to paste a JSON array of questions.
- A "Publish Quiz" button.
- When clicked, save to a Firestore collection called `quizzes`.
- On success, display a shareable link: `mysite.com/index.html?quizId=[DOC_ID]`.
- Also display a link to the dashboard: `mysite.com/dashboard.html?quizId=[DOC_ID]`.

### 3. The Student App (`index.html` & `script.js`)
- On page load, read `quizId` from `window.location.search`.
- If no `quizId` exists, show an error UI: "Invalid Quiz Link".
- If it exists, fetch the quiz document from the `quizzes` collection. Update the welcome screen with the `title` and configure the timer.
- The login form should only ask for "Name" and "Student ID" (no hardcoded validation).
- When the student submits, push an object to the `quizzes/{quizId}/submissions` sub-collection containing: `studentName`, `studentId`, `score`, `tabSwitches`, and `timestamp`.

### 4. The Teacher Dashboard (`dashboard.html` & `dashboard.js`)
- Reads the `quizId` from the URL.
- Queries the `quizzes/{quizId}/submissions` sub-collection.
- Displays a clean data table showing: Student Name, ID, Score, Tab Switches, and Time of Submission.

### Constraints
- Strictly Vanilla JS and standard DOM manipulation. No React/Vue.
- Use Firebase V9 Modular syntax (`import { getFirestore, doc, setDoc, collection, addDoc, getDocs } from "firebase/firestore"`).
- Maintain the dark glassmorphism CSS theme across all new pages.
```
